# MV Generate PRD

## 概述

MV Generate 是一个个人使用的歌词视频生成工具，提供 Web 界面和 CLI 两种使用方式。用于将 MP3 音频 + 纯文本歌词 + 封面图片自动合成为带有酷炫视觉效果的歌词视频，适用于 TikTok 等短视频平台发布。

## 输入与输出

### 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `--audio` | MP3 文件 | 是 | 歌曲音频 |
| `--lyrics` | 文本文件 | 是 | 歌词，支持 Suno 格式（自动清洗） |
| `--cover` | 图片文件 | 是 | 封面图片（JPG/PNG） |
| `--aspect` | 字符串 | 否 | 画面比例，`9:16`（默认）或 `16:9` |
| `--theme` | 字符串 | 否 | 视觉主题，默认 `neon`，可选 `vinyl` / `wave` |
| `--lyrics-style` | 字符串 | 否 | 歌词显示方式，默认 `karaoke`，可选 `fade` / `word-fill` |
| `--output` | 文件路径 | 否 | 输出文件路径，默认 `output.mp4` |

### 输出

- MP4 视频文件
- 分辨率：1080x1920（9:16）或 1920x1080（16:9）
- 帧率：30fps
- 视频编码：H.264
- 音频编码：AAC
- 时长：与整首歌曲一致

### 使用示例

```bash
python mvgenerate.py \
  --audio song.mp3 \
  --lyrics lyrics.txt \
  --cover cover.jpg \
  --aspect 9:16 \
  --theme neon \
  --lyrics-style karaoke
```

## 模块详细设计

### 1. 歌词预处理模块

**职责**：在对齐之前清洗歌词文本。

**处理规则**：
- 去除空行（合并多余空行）
- 去除段落标记：删除 `[Verse]`、`[Chorus]`、`[Bridge]`、`[Intro]`、`[Outro]` 等 Suno 风格标识（匹配 `[...]` 格式）
- 每行 trim 前后空白
- 过滤后只保留有效歌词行

**示例**：

```
输入：
[Verse]
月光洒在窗台上
你的影子在摇晃

[Chorus]
我们一起唱吧
这首歌属于夜晚

清洗后：
月光洒在窗台上
你的影子在摇晃
我们一起唱吧
这首歌属于夜晚
```

### 2. 歌词对齐模块

**职责**：使用 Whisper 将歌词与音频进行时间对齐。

- 使用 `openai-whisper` 或 `faster-whisper` 进行强制对齐（forced alignment）
- 输入：MP3 + 清洗后的纯文本歌词
- 输出：每句（及每字）的起止时间戳
- 流程：先用 Whisper 转录音频，再将用户提供的歌词与转录结果做对齐匹配
- 中文歌曲使用 `large` 模型以保证精度

### 3. 音频分析模块

**职责**：提取音频特征，为可视化提供数据。

- 使用 `librosa` 提取：
  - **频谱数据**：用于柱状/环形音量可视化
  - **节拍检测**：用于光盘旋转速度、脉冲效果同步
  - **RMS 音量**：用于呼吸缩放等动画幅度控制
- 按帧率（30fps）采样，生成逐帧的音频特征数组

### 4. 视频渲染模块

**职责**：逐帧生成视频画面。

- 使用 `Pillow` 或 `cairo` 逐帧生成画面
- 每帧根据当前时间点，组合以下图层：
  - 背景层（渐变/粒子/波浪）
  - 光盘层（旋转封面）
  - 音量可视化层（频谱条/波浪）
  - 歌词层（文字渲染）
- 帧率：30fps
- 分辨率：1080x1920（9:16）或 1920x1080（16:9）

### 5. 合成输出

**职责**：将帧序列与音频合并为最终视频。

- 使用 FFmpeg 将帧序列 + MP3 合并为 MP4
- 视频编码：H.264
- 音频编码：AAC

## 视频结构时间线

```
[0s ~ 3s]       封面展示（封面图 + 歌曲名 + 艺术家名，淡入）
[3s ~ 5s]       过渡动画（封面缩小为光盘，UI 元素就位）
[5s ~ 结束-3s]  主体播放（旋转光盘 + 音量可视化 + 歌词显示）
[结束-3s ~ 结束] 结尾淡出
```

## 三种视觉主题

### 主题 1：「霓虹脉冲」(neon) — 默认

- **背景**：深色渐变（深蓝 -> 深紫），带微弱粒子浮动
- **封面展示**：封面图片居中，带霓虹色发光边框
- **光盘**：封面图缩为圆形旋转光盘，外圈有霓虹光环脉动
- **音量可视化**：底部环形频谱条，颜色随节拍在青色/粉色/紫色间渐变
- **歌词**：白色文字，当前句带霓虹发光效果
- **整体感觉**：科技感、TikTok 风格

### 主题 2：「极简唱片」(vinyl)

- **背景**：纯色或柔和渐变（米白/浅灰/莫兰迪色调）
- **封面展示**：封面图片居中，带轻微阴影
- **光盘**：模拟黑胶唱片质感，封面嵌在唱片中心，可见唱针臂
- **音量可视化**：底部简约柱状波形，单色（黑或深灰）
- **歌词**：深色无衬线字体，简洁淡入淡出
- **整体感觉**：高级感、Apple Music 风格

### 主题 3：「律动波浪」(wave)

- **背景**：深色，带缓慢流动的波浪纹理
- **封面展示**：封面图片居中，圆角矩形
- **光盘**：封面图带呼吸缩放效果（随节拍微微放大缩小）
- **音量可视化**：光盘周围的环形波浪线，随音乐律动起伏
- **歌词**：白色文字，逐句从下方滑入
- **整体感觉**：动感、沉浸

## 三种歌词显示方式

### 方式 1：「KTV 逐句高亮」(karaoke) — 默认

- 屏幕上同时显示 3-5 句歌词（上下文可见）
- 当前正在演唱的那句高亮（放大 + 亮色），其余灰显
- 随歌曲推进，整体平滑滚动

### 方式 2：「逐句淡入」(fade)

- 每次只显示当前一句歌词
- 前一句淡出，当前句淡入，居中显示
- 简洁，适合歌词较短的歌曲

### 方式 3：「逐字填充」(word-fill)

- 显示当前句完整文字，但初始为灰色
- 随演唱进度，每个字逐步被高亮填充
- 类似 Apple Music 歌词效果，最精细
- 需要 word-level 时间戳，对中文精度可能稍差；如果对齐不够准确，自动降级到方式 1（karaoke）

### 功能说明

- **上传区**：支持拖拽或点击上传 MP3、歌词文件（.txt）、封面图片（.jpg/.png）；上传封面后即时预览
- **配置区**：单选按钮选择画面比例、视觉主题、歌词显示方式；均有默认值
- **歌曲信息**：输入歌曲名和艺术家名（用于封面展示阶段显示）
- **生成按钮**：三个必填文件上传后激活；点击后禁用，防止重复提交
- **进度条**：通过 SSE 实时推送进度（歌词对齐中 / 音频分析中 / 渲染帧 N/M / 合成中）
- **结果区**：生成完成后显示视频预览播放器 + 下载按钮

### API 设计

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/upload` | POST | 上传文件（MP3/歌词/封面），返回文件 ID |
| `/api/generate` | POST | 提交生成任务，返回 task ID |
| `/api/progress/{task_id}` | GET (SSE) | 实时推送生成进度 |
| `/api/result/{task_id}` | GET | 获取生成结果（视频文件下载） |
